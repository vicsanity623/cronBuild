How It Works

CRON fires → Workflow starts → Checkout repo ↓ Install opencode + authenticate gh ↓ Read MEMORY.md → "Day 3. Next: Build Forge System" ↓ Launch AI Agent with full project context ↓ ┌─────────────────────────────────────┐ │ Agent implements target feature │ │ git branch → commit → push │ │ gh pr create → gh pr merge │ │ Update MEMORY.md with next target │ └─────────────────────────────────────┘ ↓ Push to main → Wait for next CRON tick 🔁

Three-Tier AI Cascade

Never gets stuck. Each tier can use a different model. If one is rate-limited or down, the next kicks in automatically.

🥇 Tier 1

OpenRouter — 200+ models (Claude, GPT-4, Gemini...). One API key for everything.

🥈 Tier 2

Google Gemini — Generous free tier. Fallback when Tier 1 is rate-limited.

🥉 Tier 3

Local Ollama — Offline, unlimited. Runs on your machine, zero API cost.

Install in 2 Minutes

Add the workflow file

Create .github/workflows/cronbuild.yml in your repo with this content:

Download raw file →name: CRONBUILD
on:
  schedule:
    - cron: '0 */6 * * *'
  workflow_dispatch:
    inputs:
      target:
        description: 'Override NEXT target'
        required: false

permissions:
  contents: write
  pull-requests: write

jobs:
  build:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install opencode
        run: curl -fsSL https://opencode.ai/install.sh | bash

      - name: Run autonomous build
        env:
          OPENROUTER_API_KEY: ${{ '${{ secrets.OPENROUTER_API_KEY }}' }}
          GOOGLE_GEMINI_API_KEY: ${{ '${{ secrets.GOOGLE_GEMINI_API_KEY }}' }}
          GITHUB_TOKEN: ${{ '${{ secrets.GITHUB_TOKEN }}' }}
        run: |
          LAST_DAY=$(grep -o '\[DAY [0-9]*\]' MEMORY.md | tail -1 | grep -o '[0-9]*' || echo "0")
          NEXT_DAY=$((LAST_DAY + 1))
          LAST_TARGET=$(grep '| NEXT:' MEMORY.md | tail -1 | sed 's/.*| NEXT: //')
          MODEL="opencode/deepseek-v4-flash-free"

          PROMPT="You are an autonomous developer.
    Goal: Day $NEXT_DAY. Target: $LAST_TARGET.
    Implement, branch, PR, merge, update MEMORY.md. AUTO-APPROVE."

          opencode run "$PROMPT" --model "$MODEL" --auto

Add your API key as a GitHub Secret

Go to your repo → Settings → Secrets and variables → Actions → Add these secrets:

  • OPENROUTER_API_KEY — Get one free (supports 200+ models)
  • or GOOGLE_GEMINI_API_KEY — Free tier

Create MEMORY.md and DIRECTIVES.md

These tell the AI what to build and how to behave. Templates in the CRONBUILD repo.

Trigger it

Push your changes. The workflow runs on schedule, or you can manually trigger it from the Actions tab. Watch your repo build itself.

Example MEMORY.md

# PROJECT MEMORY LOG
*Format: [DAY X] | [DATE] | [COMPLETED] | [NEXT DAY TARGET]*

[DAY 0] | 2026-07-14 | Initial setup — repo initialized with
CRONBUILD autonomous pipeline. | NEXT: User Authentication —
Add login/signup pages, JWT tokens, password hashing, and
session management in a new auth module.

The AI autonomously writes its own NEXT targets by auditing your codebase after each build cycle. No manual editing needed — but you can override any time.
